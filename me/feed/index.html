Then this post is made for you!

Introduction

Map on WordPress Wix and others.

If I tell you: website, you may think WordPress, Wix or Blogger…

But if you have a blog or a simple informative site you don’t need such powerful tools with so many features to load on it!

You can simply use a static site….

A static site is roughly speaking a site using only HTML and CSS files.

To create one you can do it, either by writing yourself the HTML code or by using a generator like Jekyll….

But if you really want simplicity, you can use WordPress too!

I will explain you how to create a static WordPress Website locally for free, how to convert it into static website and then uploading it on GitHub Pages.

How it works?

To make this we will use WordPress to create a local website and then convert it into HTML files.

For this we will use: Wget, XAMPP, WordPress, A theme, Git, GitHub Pages…

I will not explain how to install Wget and Git here but this is the install command on Ubuntu:

sudo apt install wget git

But you can find easily how to install them on your Operating System.

Setup XAMPP

First, go on this link and download the file on your computer in new directory named “wordpress”.

If you are on Linux you can use AIP to install the file or just run the following commands:

chmod +x *.AppImage
sudo ./*.AppImage

Then click all the time on “next” to proceed to the installation…

XAMPP Configuration

Once the XAMPP installation finished, click on “finish” to start XAMPP, then switch tab to Manage Servers and start Apache and MySql Database…

Then you will need to have a primary access to files. On Linux just run the following commands:

cd /opt/lampp/htdocs
sudo chmod 777 -R .
sudo chmod 775 phpmyadmin/config.inc.php

Now you have downloaded the WordPress archive place it in your “wordpress” directory and extract his content.
Then go in /opt/lampp/htdocs in a new window. Copy the archive files into htdocs and replace “index.php”.

Now you can test you XAMPP setup by going to the following link:

http://localhost

If you see something like this, it works:

MYSQL Configuration

Then go on the PhpMyAdmin Pannel and click on new, write the following settings:

Then click on create.

We just created a local database, now we need to create a user to use it.

Click on “localhost” then click on Users then New

Finally tick the box “Check all” and lick on “go”.

WordPress Setup

Now, come back on http://localhost, click on “next”check your information and click again on next.

Now you can fill your personal information, there are mine:

Once you finished click on “Start WordPress”…

WordPress Configuration

Then you can login with your information and be on you dashboard.

Next, go in wp-config.php and add the following line to it:

define( 'FS_METHOD', 'direct' );

Then press CTRL + S and run the following command:

sudo chmod 777 -R .

Next go on your dashboard, click on Apparance, add new, upload theme and drag-and-drop your theme on it.

Then click on “activate”.

Finally go to customize and change the things you want to customize from the base theme.

Then create some pages and posts by clicking on “+”

Once you finished to write, un-tick the “allow comments” section if it is activated and change the different settings on this side…

Once you completely finished this first version of your website we will need to convert it into static website…

Converting into static website using Wget

To convert your website into a static website copy-paste the following comand:

wget --mirror --convert-links --adjust-extension --page-requisites --no-parent http://localhost

Now you can go to the “localhost” directory and test your static website by going to “index.html”.

Now we will upload it on GitHub Pages but you can skip this part and upload it on your FTP server if you have one…

Upload to GitHub Pages

To upload it we will use the GitHub Pages system. To make this create a new repository and name it as YOURUSERNAME.github.io

Then open “localhost” as terminal and start the following commands:

git init
git remote add origin https://github.com/USERNAME/USERNAME.github.io
git add –all
git commit -m “website”
git push -u origin master

Then you can connect your terminal with your GitHub account. Wait some minutes to be uploaded and changed.

Demo

To test if your website has been uploaded just go to the following URL:

YOURUSERNAME.github.io

If all’s right you should see your website there!

Linux Tip

Little Linux exclusive tip: this is a little script to reduce the number of commands to run to update your website

Just copy-paste the following code in a file named install.sh:

sudo echo "cd ~/PATH/TO/wordpress" > /usr/bin/wp-start
sudo echo "sudo /opt/lampp/lampp start" >> /usr/bin/wp-start
sudo echo "echo Go to http://localhost/wp-admin to access to your website..." >> /usr/bin/wp-start
sudo chmod 777 /usr/bin/wp-start

sudo echo "cd ~/PATH/TO/wordpress" > /usr/bin/wp-stop
sudo echo "wget --mirror --convert-links --adjust-extension --page-requisites --no-parent http://localhost" >> /usr/bin/wp-stop
sudo echo "cd localhost" >> /usr/bin/wp-stop
sudo echo "git add --all" >> /usr/bin/wp-stop
sudo echo "git commit -m "website"" >> /usr/bin/wp-stop
sudo echo "git push -u origin master" >> /usr/bin/wp-stop
sudo echo "sudo /opt/lampp/lampp stop" >> /usr/bin/wp-stop
sudo chmod 777 /usr/bin/wp-stop

Then replace the following values:

  • PATH/TO is the path to the folder “wordpress” who contain the localhost directory…

If you want you can replace the “git” commands by the FTP uploads commands.

To install this file start the following command:

sudo chmod +x install.sh && sudo sh install.sh

Starting this point when you want to update your website just start: “wp-start”, make your modifications and when you finished run “wp-stop”

This script will manage the starting process of XAMPP, the Upload and the stop of XAMPP in only 2 commands.

Conclusion

So now you know how to create a static website using WordPress, installing WordPress locally, using Wget, uploading to GitHub Pages and create an automation script of the update!

Thanks for your attention!

]]> Install any app in one command on Ubuntu (Linux) http://localhost/me/2019/07/08/install-any-app-in-one-command-on-ubuntu-linux/ Mon, 08 Jul 2019 18:47:09 +0000 http://localhost/?p=44 Imagine being on your favorite Linux distro with 10 different installers to run with 3 commands for each. That is not a good experience of course but I will explain you how to install all deb, AppImages, sh, Flatpak and some exes in only one command.

I called it AIP it’s Open Source (of course) and you can install it with the following commands:

git clone https://github.com/chopin2712/Auto-Installer-process
cd Auto-Installer-process
sudo sh setup.sh

Once you installed it you can go in the directory who contain the installer and open it in terminal.

Then run the following command:

sudo aip

Then your file will be installed and then removed (except for the .jar and .AppImage files formats who are exceptions because if you remove them you remove the software…)

Don’t have installer(s)?

If you know the package name but you don’t have the installer, there is another thing you can do with AIP.

Just run the following command:

sudo aipn

Then type the package name

Once you see in the terminal the app is installed, stop the script by pressing CTRL + D to prevent the conflicts.

Then your app will be installed if your package name is valid.

Want to uninstall?

There is no functions to uninstall the apps, but you can do it using Synaptic by running the following command:

sudo synaptic

Then you can search your package, check the box and choose complete removal.

Click on Apply and then Confirm to uninstall the package(s).

Want to contribute?

You can get the source code on GitHub on the following link:

https://github.com/chopin2712/Auto-Installer-Process

Don’t works properly?

You can post an issue on the GitHub project to get support or make a feature request. Thanks for using this app.

This is my real first post on this website. I will explain you how to use my latest creation. It is called MineCraft Server Setup (MCSS) and is an MC simple server installer. Only linux (for the moment)

It’s made with Python and it’s Open Source. You can browse the source on this link. I am very proud of this! It supports Vanilla, CraftBukkit and Spigot at the moment. This is a little tutorial to explain you how to create your very own MC server.

First run the following commands:

git clone https://github.com/chopin2712/MCSS
cd MCSS
chmod +x install.sh
sudo sh install.sh

Then create a new directory on your computer and open it in terminal…

The real magic happen here, you have just to run:

sudo MCSS

First answer 1 for Vanilla or 2 for craftbukkit/spigot.

Second, download the server.jar file on https://mcversions.net , put it in the directory then press enter.

Third, if you want to change the basics settings of Minecraft answer “Y” and modify your settings (TIPS: For online mode change into false to accept cracked versions)

For the “world wide step”, I recommend you to answer “N” to skip this part for the moment because this feature is not stable enough to be used successfully…

Finally your server will start, just wait 20 seconds and you can follow the above IP addresses to join your world.

You can write “stop” to shut down your Minecraft Server and you can run “op USERNAME” to be admin on your server and run all the admins commands on it directly on Minecraft.

It is very simple to use in deed and my future ideas for this app are:

  • Making the world wide feature works perfectly
  • Making available cross-platform
  • Possibly trying to make a visual GUI to use the app (far in the future)
  • Etc, etc

You can contribute and access to this project by clicking on the following link:

https://github.com/chopin2712/MCSS

Already installed? You can update your version to a higher one by running the following script:

cd MCSS
sudo sh install.sh

NB: If you made a server in a older version, the update don’t have any effect and your server will works properly…
